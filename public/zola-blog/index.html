Init

Get

brew install zola

Generate

zola init codingezio.github.io
zola serve

Theme

Get

git \

    themes/lightspeed

Override

# Required
cp -rf themes/lightspeed/templates/* templates/
cp -rf themes/lightspeed/public/*    public/
cp -rf themes/lightspeed/sass/*      sass/


# Don't worry, you can add/edit/remove these .md files later
cp -rf themes/lightspeed/content/*   content/

Submodule

Put these in a file named .gitmodules

[submodule "themes/lightspeed"]
    path = themes/lightspeed
    url = https://github.com/carpetscheme/lightspeed.git

Publish

Token

  1. Go to Personal access tokens
  2. Click Generate new token
  3. Add your basic info like Note and Expiration
  4. Scroll to Select scopes

    Check the root level of repo and workflow on, it works for me though

  5. Save it as it would only show once

GitHub Action

Set Secret
  1. Go to Security -> Secrets -> Actions under the [Settings tab]
  2. Click New repository secret
  3. Add your basic info
    • Name: it would be of use when you write the GitHub Actions Workflow
    • Secret: the token you generated in the Personal access tokens page
Configure

  1. Go to the Actions tab of your GitHub Pages repo

  2. Click New Workflow

  3. Click set up a workflow yourself

  4. Copy and edit this into your main.yml

    Do read my comments inside

    # On every push this script is executed
on: push
name: Build and deploy GH Pages
jobs:
  build:
    runs-on: ubuntu-latest
    # Change 'master' to 'main' depends on your main branch name
    if: github.ref == 'refs/heads/master'
    steps:
      - name: checkout
        uses: actions/checkout@v2
      - name: build_and_deploy
        uses: shalzz/zola-deploy-action@v0.14.1
        env:
          # You build the blog files in 'master' (or 'main')
          # It spits out to the 'gh-pages' branch (GitHub Pages)
          PAGES_BRANCH: gh-pages
          # The name of the secret you set under this repository
          # Go to 'Security -> Secrets -> Actions' to check :)
          TOKEN: ${{ secrets.YOUR_TOKEN_NAME }}

  5. Check Pages settings, make sure that

    • The Source shall be Deploy from a branch
    • The Branch shall be gh-pages and /(root (remember to click save)

Deploy

Now go back to your local repository
