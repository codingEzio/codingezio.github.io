要讨论的话题

  • 索引与约束: 高效找数据 + 入库前先精筛
  • 事务：让数据符合预期的存入数据库 + 全入或不入逻辑
  • 并发控制：较代码稍高一层的抽象，处理冲突 (i.e. 锁)
  • 分库分表：配合并发；将数据分至多节点，分摊单节点压力
  • 集群：配合并分库表；分担压力顺便高可用，或备份，或恢复
  • 优化：SQL 本身，运行 SQL 的软件 (比如从配置入手)
