The Quality of Work I Wanna Achieve

Posted on November 04, 2019

These are some principles for myself to follow

I'd try/like/love to

  • write readable code -- typing annotations + mypy, meaningful variable name ..
  • write scalable code -- seperating into small functions even it's achieves little ..
  • respect linter/formatters' rules -- hardcore on 'typingOnFormat'(in VSCode)
  • write no-basic-grammer-errors English documents -- no broken English, no Chinglish
  • write more-native English documents -- so that the code we write would have a bigger impact (private and open-sourced projects)
  • respect others' work -- no offsensive words against mangers/co-workers ever
  • write documents that provides context -- documents/comments should act like a story (at lease for part of the code?)
  • no politics during work
  • actually implement TDD approach -- I'll try my best since I havn't used to it right now

On programming, I'd prefer

  • use for loop instead of list comprehensions in Python (it depends!!) (more on this)

  • add (not-too-much)(probably)meaningless/verbose spaces instead of compressing spaces to save so-called space/efficiency

On personal habits, I'd prefer

  • read mutliple books at the same time instead of just sticking to one (which might result into lack of focus, or not)
  • write blogs in plain .txt or .md then push to Github instead of deploy as static site (why? please see references)

What I want (well, if, IF I get to choose 😅)

  • co-workers that respect each other's space
