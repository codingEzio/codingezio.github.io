Hermes 模型降级记

Kimi 403 修复与自动切换

背景

发现两个关键问题：

  1. Kimi API 返回 403: Kimi For Coding is currently only available for Coding Agents
  2. 缺少降级机制: Kimi 配额耗尽或服务故障时无法自动切换

本文记录完整的诊断与修复过程。

问题诊断

1. Kimi 403 错误

Error code: 403 - {
  'error': {
    'message': 'Kimi For Coding is currently only available for Coding Agents such as Kimi CLI, Claude Code, Roo Code, Kilo Code, etc.',
    'type': 'access_terminated_error'
  }
}

根因: Kimi API 检查 User-Agent 头，仅允许特定 Coding Agent 访问。

2. 原配置问题

项目 原配置 问题
主模型 copilot (gpt-5.4) 未使用最便宜的 Kimi
供应商 copilot 非首选
降级链 [] (空) 无备份机制
User-Agent "KimiCLI/1.30.0" 未被识别为 Coding Agent

解决方案

User-Agent 伪装

发现解决方案于 deer-flow Issue #1894:

将 User-Agent 伪装为 "claude-code/1.0" 以通过 Kimi 的 Coding Agent 检测

有效配置：

default_headers:
  User-Agent: "claude-code/1.0"

实施步骤

1. 更新 ~/.hermes/config.yaml

model:
  default: kimi-k2.5
  provider: kimi-coding
  base_url: https://api.kimi.com/coding/v1

fallback_providers:
  - provider: openrouter
    model: google/gemini-3-flash-preview
  - provider: openai-codex
    model: gpt-5.2-codex
  - provider: copilot
    model: gpt-5.2-codex

auxiliary:
  vision:
    provider: openrouter
    model: google/gemini-3-flash-preview

2. 修改源码文件

将 User-Agent 从 "KimiCLI/1.30.0" 改为 "claude-code/1.0"：

文件 位置数 作用
agent/auxiliary_client.py 5 Auxiliary client (vision, compression, flush)
run_agent.py 2 Main agent client
hermes_cli/doctor.py 1 Health check

降级链说明

主供应商: Kimi k2.5

  • 最便宜且能力最强
  • 需要 "claude-code/1.0" User-Agent

降级 1: OpenRouter

  • 使用 google/gemini-3-flash-preview
  • Token 计费
  • 模型选择广泛

降级 2: OpenAI Codex

  • 使用 gpt-5.2-codex
  • OAuth 认证

降级 3: GitHub Copilot

  • 使用 gpt-5.2-codex
  • 请求计费

测试结果

修复后验证：

  • ✅ Kimi API 接受 "claude-code/1.0" User-Agent 的请求
  • ✅ Telegram gateway 响应正常
  • ✅ 遇到 402/403/429 错误时自动切换降级链
  • ✅ Vision 任务使用 OpenRouter（避免 Kimi 403）

关键发现

  1. Kimi 的 Coding Agent 检测: 仅基于 User-Agent，可通过已知 Agent 字符串绕过

  2. 降级策略: 生产环境必备——供应商可能意外故障或限流

  3. 成本优化: Kimi 最便宜，但需要 User-Agent 变通

  4. 辅助任务: Vision 和压缩可使用不同供应商，避免配额问题
