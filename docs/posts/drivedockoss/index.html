DriveDockOSS: 弹出磁盘之前，先看清楚是谁还在用它

macOS 告诉你“磁盘正在使用中”时，真正缺少的不是另一个警告，而是一个名字：哪个 app、哪个进程、哪条路径还没有放手？

DriveDockOSS 是一个很小、也很专一的原生 macOS app。它不想接管你的磁盘管理，只想在你拔掉硬盘之前，把最关键的信息摆到眼前。

一眼看到重点

DriveDockOSS 真正的主界面是 menu bar 里的 popover。点一下图标，就能同时看到磁盘的剩余空间、连接方式、Time Machine 状态，以及仍在使用它的 app 和文件句柄。

如果 DemoDrive 还被 SyncAgent 或 VideoEdit 占用，你不需要先尝试弹出、等系统失败，再猜到底该关什么。答案已经在 popover 里了。

需要展开时

菜单栏适合快速看一眼；需要比较多个磁盘、查看更多信息时，可以打开可调整大小的 desktop 窗口。同一份磁盘状态会被展开，而不是换成另一套难以对应的界面。

找到具体是谁

选中某个磁盘后，句柄详情会列出进程名、PID、读写方式和完整路径。重点不是告诉你一个抽象的“正在使用”，而是让你知道该回到哪个 app、哪个文件，处理完什么再弹出。

它不会替你蛮干

DriveDockOSS 会在确认后请求正常弹出，也可以向普通 GUI app 发送正常退出请求，但不会强制退出进程或强制弹出磁盘。虚拟机和容器宿主仍然需要按它们自己的方式关闭。

它也不会把“没有看到句柄”假装成绝对安全。受系统权限保护的活动可能不会被 lsof 显示；这个工具减少猜测，但不会替你做过度自信的判断。

0.1.1

当前版本支持 Apple silicon 和 macOS 15.6.1 以上系统，界面提供英文、简体中文、繁体中文、韩文和日文。下载包使用 ad-hoc 签名且尚未 notarize，第一次打开时 macOS 可能要求手动批准。

你可以直接下载 DriveDockOSS 0.1.1，也可以从源码构建。它想解决的始终是一个很小、但会反复出现的问题：拔盘之前，把猜测变成看得见的信息。
