DriveDockOSS: 弹出磁盘之前，先看清楚是谁还在用它

macOS 告诉你“磁盘正在使用中”时，真正缺少的不是另一个警告，而是一个名字：哪个 app、哪个进程、哪条路径还没有放手？

DriveDockOSS 就是为这个问题做的。它把磁盘信息、可见的文件句柄和 Time Machine 状态放在同一个原生 macOS 界面里，让弹出磁盘之前的判断少一点猜测。

它做什么

  • 显示内置和外接磁盘的容量、型号、连接方式与 SMART 信息；
  • 列出 lsof 能看到的进程、PID、读写模式和完整路径；
  • 标出 Time Machine 目标磁盘与正在进行的备份；
  • 在确认后请求正常弹出，或向普通 GUI app 发送正常退出请求。

它不提供强制退出，也不提供强制弹出。虚拟机、容器宿主和受权限保护的活动仍然需要用户自己判断。这不是少做一个功能，而是产品的安全边界。

为什么另开一个 OSS repo

原来的 DriveDock 是私人项目，里面有自己的历史、工作区约定和机器环境。开源版没有直接搬运那段历史，而是从经过审阅的文件清单开始，重新建立一个干净仓库。别人拿到的是 app 代码、构建脚本、测试、README、贡献说明和 License，不是我的本机状态。

0.1 版本

首个版本支持 Apple silicon 和 macOS 15.6.1 以上系统，界面提供英文、简体中文、繁体中文、韩文和日文。当前下载包使用 ad-hoc 签名且未 notarize，第一次运行时 macOS 可能要求手动批准；也可以直接按照 README 从源码构建。

这只是第一个公开版本。README 和 Release 会继续保持简单、可读；以后如果有更多开源 app，再让它们进入一个独立的应用展示站，而不是把每个项目都塞进一篇长文章里。
