Hermes: Kimi 403 Fix

Kimi 接入 Hermes 的 403 修复

先说结论

默认情况下，Kimi for Coding 不会把 Hermes 当成它认可的 Coding Agent。 若直连，最常见的结果就是 403。

要把它接上，真正关键的只有两处：

  1. base_url 指向 https://api.kimi.com/coding/v1
  2. default_headers 里带上它能接受的 User-Agent

最小配置

model:
  default: kimi-k2.5
  provider: kimi-coding
  base_url: https://api.kimi.com/coding/v1

default_headers:
  User-Agent: "claude-code/1.0"

这些改法是怎么知道的

线索其实来自两边：

  1. Kimi 官方文档：能确认 Coding 接口入口就是 /coding/v1
  2. 外部问题讨论：能看到 403 现象，以及 User-Agent 是否会影响放行

把这两边合起来看，就会知道这件事重点不在“模型名写没写对”，而在 入口地址 + 请求头。
