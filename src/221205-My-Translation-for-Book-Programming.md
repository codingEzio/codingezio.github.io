## Idea 主意

### What 细节

> Back then I have quite a lot prejudices against websites like Zhihu, Juejin. Well, I still do ☺️! But now I want and need to find a tool that could compile these Markdown files into a book/documentation-like format, so that people could navigate and read this book more easily.
>> 曾经我对知乎、掘金这样的网站很反感。我的看法依然未变！但是为了优化读者的体验，我现在正在找一样工具可以将这些文件，汇编为类似书籍或技术文档的格式。

> A static-site generator [Docusaurus](https://docusaurus.io/docs) which is built by *Facebook* caught my eye. It looks really good both for the layout and the flow of navigation. My choice would be this one for now. Also, this shall be in companion with another tool, so the time and effort of building and deploying could be saved for me. [Cloudflare Pages](https://developers.cloudflare.com/pages/) looks pretty good. I would probably eventually settle on this one, in comparison with similar free alternatives like [Surge](https://surge.sh/pricing), [Netlify](https://www.netlify.com/pricing/) and [Vercel](https://vercel.com/pricing).
>> 由 Facebook 公司主导开发的 [Docusaurus](https://docusaurus.io/docs) 静态网站生成器看起来蛮有意思的，外观和不同内容之间的导航都很舒服。估计我最终的选择就是这个。当然，我还要有另外一样能够帮助我构建并且部署的省时省力的工具。比起来各类免费的替代选择，比如 [Surge](https://surge.sh/pricing)、[Netlify](https://www.netlify.com/pricing/) 以及[Vercel](https://vercel.com/pricing)，[Cloudflare Pages](https://developers.cloudflare.com/pages/) 看起来更胜一筹，这应该是我的最终选择。
