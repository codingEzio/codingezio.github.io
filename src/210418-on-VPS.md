> VPS: Virtual Private Server

### Keep Process Running at Background

> Even after ending the SSH session. Like [how to keep processes running after ending ssh session](https://askubuntu.com/a/8657/1650652)

```bash
# There were also other options like nohup, tmux etc.

# Run 'screen' before you wanna do anything

# 1. if you only need one task being executed at the background
screen               # enter the environment
# [ctrl+A], [ctrl+D] # exit the environment
screen -r            # re-enter the environment

# 2. if you need to run multiple tasks at the background
screen -S task1      # name the env as 'task1' then enter the env
# [ctrl+A], [ctrl+D] # exit the environment
screen -r task1      # re-enter the environment named 'task1'
```

### Synchroize Files using `rsync`

```bash
# Transfer files FROM local TO your server
# $ rsync -v -e ssh /www/backup.tar.gz jerry@openbsd.nixcraft.in:~
# $ rsync -v -e ssh ~/.zshrc root@1.2.3.4:~
rsync -v -e ssh PATH_TO_DIR_OR_FILE USERNAME@SERVER_IP:SERVER_DIR_PATH

# Transfer files FROM your server TO your local machine
# $ rsync -v -e ssh jerry@openbsd.nixcraft.in:~/webroot.txt /tmp
# $ rsync -v -e ssh root@1.2.3.4:/root/dl_vids/ ~/Desktop/vids/
rsync -v -e ssh USERNAME@SERVER_IP:PATH_TO_DIR_OR_FILE LOCAL_DIR_PATH
```

### Increase SSH Connection Timeout

> [Increase SSH Connection Timeout in Linux](https://web.archive.org/web/20200909024011/https://www.tecmint.com/increase-ssh-connection-timeout/)

```bash
# ClientAliveInterval 3600
# ClientAliveCountMax 3
vim /etc/ssh/sshd_config

# Reload
systemctl reload sshd
```
