## Prologue

- Web *R*eal *T*ime *C*ommunication
- Enable media <small>(video/music)</small> streaming <small>(transporting)</small> between clients <small>(e.g. watching streams on Twitch.tv)</small>.
- Enable browsers to stream files <small>(like videos)</small> directly to one another <small>(less cases in using server-side hosting)</small>
- Back then, if you wanna do *RT* <small>(big files (produced by vid call))</small> *C* <small>(normally vid call)</small>
  - you need to use proprietary softwares like *Skype*
  - you no longer need to, as we have a project (with its APIs) to accomplish that
  - you can use it on [almost every browsers](https://en.wikipedia.org/wiki/WebRTC#Support)

## Technically Speaking

- They <small>(WebRTC and [its alternatives](https://restream.io/blog/streaming-protocols/))</small> were all application protocols built on *UDP* and/or *TCP*
- A consensus between different browsers on how to send and receive the data
- A way for the data to transport more efficiently
